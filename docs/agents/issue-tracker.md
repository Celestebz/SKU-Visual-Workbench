# Issue Tracker

This project uses local markdown as its issue tracker.

- PRDs live under `.scratch/frontend-workbench/`.
- Implementation issues live under `.scratch/frontend-workbench/issues/`.
- Issues should be written in dependency order.
- Each issue should be independently grabbable by an agent and include acceptance criteria.

There is no external PR triage surface for this local tracker. If the project later moves to GitHub, these markdown issues can be migrated into GitHub Issues.
